## Protobuf의 사용 목적과 통신 계층

- Protobuf는 데이터를 **작고 빠르게 직렬화하기 위한 포맷**이며, 일반적으로 [[./http-basic|HTTP]], WebSocket 등 **기존 통신 계층 위에서 사용되는 구조**  
- 통신 계층은 여전히 TCP / HTTP / [[./https-basic|HTTPS]] 등이며, Protobuf는 그 위에서 **payload 최적화** 역할을 수행함  
- 성능 향상의 본질은 **데이터 구조 자체의 경량화**에 있으며, **통신 계층을 우회하거나 대체하지 않음**
